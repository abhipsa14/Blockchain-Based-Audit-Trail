--------------------------------------------------------------------------------
1.1 Why Agile Model?
--------------------------------------------------------------------------------

1. EVOLVING REQUIREMENTS
   - Project started with basic CTGAN data generation
   - Privacy verification added incrementally
   - Multiple compliance standards (GDPR, HIPAA, EU AI Act) added over time
   - Model caching added to improve performance

2. MODULAR ARCHITECTURE
   - Each component (Generator, Verifier, Blockchain) works independently
   - Allows parallel development of different modules
   - Easy to add new features without breaking existing functionality

3. CONTINUOUS FEEDBACK
   - Web dashboard provides immediate user feedback
   - Verification scores give instant quality assessment
   - Current results: Privacy 54.57%, Utility 73.27%, Bias 37.80%

4. WORKING SOFTWARE PRIORITY
   - Each sprint delivers functional components
   - Users can start using partial features immediately
   - Value delivered incrementally
