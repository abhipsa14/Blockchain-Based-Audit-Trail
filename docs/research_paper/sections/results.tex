\subsection{Discussion}

\subsubsection{Trust vs Latency Trade-off}
The blockchain approach adds $\sim$2 seconds latency but provides significantly higher trust guarantees. For datasets requiring regulatory compliance, this trade-off is acceptable.

\subsubsection{Fairness Challenges}
The low fairness scores (43.8\%) highlight a common challenge in synthetic data: GANs can amplify existing biases in training data. Future work should integrate bias mitigation during generation.

\subsubsection{Limitations}
\begin{itemize}
    \item Current implementation uses simulated blockchain
    \item k-anonymity computation is approximate for large datasets
    \item Byzantine fault tolerance limited to $<$1/3 malicious verifiers
\end{itemize}
