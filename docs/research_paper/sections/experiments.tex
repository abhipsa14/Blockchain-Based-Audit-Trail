\subsection{Experimental Setup}

\subsubsection{Datasets}
We evaluate our system on the UCI Adult Income dataset:
\begin{itemize}
    \item \textbf{Records:} 32,561 training samples
    \item \textbf{Features:} 15 attributes (mixed numerical and categorical)
    \item \textbf{Protected Attributes:} sex, race, age
    \item \textbf{Target:} income ($>$50K or $\leq$50K)
\end{itemize}

\subsubsection{Approaches Compared}
We compare three approaches:
\begin{enumerate}
    \item \textbf{Baseline:} No verification (generation only)
    \item \textbf{Centralized:} Single trusted verifier
    \item \textbf{Blockchain (Ours):} Distributed verification with consensus (3 verifiers)
\end{enumerate}

\subsubsection{Evaluation Metrics}
\begin{itemize}
    \item \textbf{Latency:} End-to-end verification time (seconds)
    \item \textbf{Trust Score:} Combined quality score (0-100)
    \item \textbf{Tamper Resistance:} Boolean (can results be modified?)
    \item \textbf{Privacy Score:} DCR + k-anonymity + MIA composite
    \item \textbf{Utility Score:} Statistical similarity + ML efficacy composite
    \item \textbf{Fairness Score:} Demographic parity + disparate impact composite
\end{itemize}

\subsection{Comparative Study}

We run each approach 10 times and measure:
\begin{itemize}
    \item Mean and standard deviation of latency
    \item Trust score distribution
    \item Quality metric consistency
\end{itemize}

\textbf{Hypothesis H1:} Blockchain-based approach will show higher trust scores due to tamper resistance.

\textbf{Hypothesis H2:} Latency overhead will be 2-3x compared to centralized, but remain under 10 seconds.

\subsection{Scalability Study}

We test system performance across two dimensions:

\subsubsection{Dataset Size Scaling}
Testing with: 1K, 5K, 10K, 50K, 100K rows

\textbf{Hypothesis H3:} Verification time scales linearly with dataset size.

\subsubsection{Verifier Count Scaling}
Testing with: 1, 3, 5, 7, 10 verifiers

\textbf{Hypothesis H4:} Consensus latency scales sublinearly with verifier count.

\subsection{Ablation Study}

We evaluate the contribution of each verification component:
\begin{itemize}
    \item Privacy-only verification
    \item Utility-only verification
    \item Bias-only verification
    \item Combined verification
\end{itemize}

\subsection{Attack Resistance}

We test system resilience against:
\begin{itemize}
    \item \textbf{Data Tampering:} Modifying synthetic data after verification
    \item \textbf{Result Manipulation:} Attempting to change verification scores
    \item \textbf{Verifier Collusion:} Byzantine verifiers submitting false results
\end{itemize}
